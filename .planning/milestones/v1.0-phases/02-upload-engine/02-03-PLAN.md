1. Create `src-tauri/src/upload/queue.rs`:

**UploadQueue struct:**
```rust
use tokio::sync::Semaphore;
use std::sync::Arc;

pub struct UploadQueue {
    /// Concurrency limiter — default 2 permits, user-configurable 1-5
    pub semaphore: Arc<Semaphore>,
    /// Current max concurrent uploads setting
    pub max_concurrent: usize,
}
```

**UploadQueue::new(max_concurrent: usize)** — creates queue with Semaphore of `max_concurrent` permits. Clamp to 1..=5 range.

**UploadQueue::set_max_concurrent(&mut self, new_max: usize)** — adjusts concurrency limit. Clamp 1..=5. If increasing: add permits. If decreasing: close the old semaphore and create a new one with fewer permits (active uploads will finish naturally).

**try_start_next** — Called after enqueue, after a job completes, or after a resume:
- Query SQLite for pending jobs ordered by `priority DESC, created_at ASC`
- Try to acquire a semaphore permit (try_acquire, not blocking)
- If permit acquired: spawn the upload worker task (for now, spawn a placeholder that logs "would start job {id}")
- The actual worker spawn will be wired in Plan 04
- Signature: `pub async fn try_start_next(queue: &UploadQueue, db: &SqlitePool) -> Option<String>` — returns job_id if started

**next_pending_job** — helper to get the highest-priority pending job:
- Query: `SELECT id FROM upload_jobs WHERE state = 'pending' ORDER BY priority DESC, created_at ASC LIMIT 1`
- Returns `Option<String>` (job_id)

**State transition helpers** (validate transitions):
```rust
pub fn can_transition(from: &str, to: &str) -> bool {
    matches!((from, to),
        ("pending", "hashing") |
        ("hashing", "uploading") |
        ("uploading", "committing") |
        ("committing", "done") |
        // Pause from any active state
        ("hashing" | "uploading" | "committing", "paused") |
        // Resume from paused goes back to pending (re-enters queue)
        ("paused", "pending") |
        // Fail from any active state
        ("hashing" | "uploading" | "committing", "failed") |
        // Retry from failed goes back to pending
        ("failed", "pending") |
        // Cancel from any non-terminal state
        ("pending" | "hashing" | "uploading" | "committing" | "paused" | "failed", "cancelled")
    )
}
```

1. Create `src-tauri/src/upload/cancel.rs`:

**register_token** — Create and store a CancellationToken for a job:
- `pub async fn register_token(tokens: &Mutex<HashMap<String, CancellationToken>>, job_id: &str) -> CancellationToken`
- Creates a new CancellationToken, clones it, stores clone in the map, returns original
- If a token already exists for this job_id, cancel the old one first, then replace

**cancel_job** — Cancel a specific upload:
- `pub async fn cancel_job(tokens: &Mutex<HashMap<String, CancellationToken>>, db: &SqlitePool, job_id: &str) -> Result<(), String>`
- Look up token in map, call `.cancel()` on it
- Update job state to "cancelled" in SQLite
- Remove token from map

**pause_job** — Pause a specific upload:
- `pub async fn pause_job(tokens: &Mutex<HashMap<String, CancellationToken>>, db: &SqlitePool, job_id: &str) -> Result<(), String>`
- Look up token in map, call `.cancel()` on it (cancellation token is reused for pause — the worker checks job state to distinguish cancel vs pause)
- Update job state to "paused" in SQLite
- Remove token from map

**pause_all** — Pause all active uploads:
- `pub async fn pause_all(tokens: &Mutex<HashMap<String, CancellationToken>>, db: &SqlitePool) -> Result<usize, String>`
- Get all tokens from map, cancel each
- Update all active jobs (state IN ('hashing', 'uploading', 'committing')) to "paused" in SQLite
- Clear tokens map
- Returns count of paused jobs

**remove_token** — Clean up token after job completes:
- `pub async fn remove_token(tokens: &Mutex<HashMap<String, CancellationToken>>, job_id: &str)`

Use `tokio::sync::Mutex` for the tokens map (same as AppState pattern).

**Hash pool initialization:**
```rust
use rayon::ThreadPoolBuilder;
use sha2::{Digest, Sha256};
use std::path::Path;
use std::sync::OnceLock;

static HASH_POOL: OnceLock<rayon::ThreadPool> = OnceLock::new();

fn hash_pool() -> &'static rayon::ThreadPool {
    HASH_POOL.get_or_init(|| {
        ThreadPoolBuilder::new()
            .num_threads(2) // cap at 2 per CONTEXT.md decision
            .thread_name(|i| format!("face-hugger-hasher-{}", i))
            .build()
            .expect("failed to build hash threadpool")
    })
}
```

**hash_file_streaming** — Stream-hash a file returning SHA-256 and file size:
- `pub async fn hash_file_streaming(path: &Path) -> Result<([u8; 32], u64), String>`
- Uses `tokio::task::spawn_blocking` bridging to rayon `hash_pool().install()`
- Reads file in 8MB buffer chunks
- Returns (sha256_hash_bytes, total_bytes_read)
- On file not found: return error "Source file not found: {path}"
- On read error: return error with OS error message

```rust
#[cfg(test)]
mod tests {
    use super::*;
    use std::collections::HashMap;
    use tokio::sync::Mutex;

    #[tokio::test]
    async fn test_register_and_cancel() {
        let tokens = Mutex::new(HashMap::new());
        let token = register_token(&tokens, "job-1").await;
        assert!(!token.is_cancelled());

        // Cancel should mark the token
        cancel_job_token(&tokens, "job-1").await;
        assert!(token.is_cancelled());
    }

    #[tokio::test]
    async fn test_cancel_nonexistent_job() {
        let tokens = Mutex::new(HashMap::new());
        // Should not panic
        cancel_job_token(&tokens, "nonexistent").await;
    }

    #[tokio::test]
    async fn test_register_replaces_old_token() {
        let tokens = Mutex::new(HashMap::new());
        let token1 = register_token(&tokens, "job-1").await;
        let token2 = register_token(&tokens, "job-1").await;
        // Old token should be cancelled
        assert!(token1.is_cancelled());
        assert!(!token2.is_cancelled());
    }

    #[tokio::test]
    async fn test_cancel_stops_select_loop() {
        let tokens = Mutex::new(HashMap::new());
        let token = register_token(&tokens, "job-1").await;

        let token_clone = token.clone();
        let handle = tokio::spawn(async move {
            tokio::select! {
                _ = token_clone.cancelled() => {
                    return "cancelled";
                }
                _ = tokio::time::sleep(std::time::Duration::from_secs(60)) => {
                    return "timeout";
                }
            }
        });

        // Cancel the token
        cancel_job_token(&tokens, "job-1").await;

        let result = handle.await.unwrap();
        assert_eq!(result, "cancelled");
    }
}
```

Note: `cancel_job_token` is a simpler helper that just cancels the token in the map without touching the DB — useful for testing. The full `cancel_job` function also updates SQLite.
