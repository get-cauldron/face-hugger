**1. src/App.tsx line 18** — Remove unnecessary `as any` cast:

Reason: `validateToken` returns `Promise<UserInfo>`, `setAuth` accepts `UserInfo`. Cast is unnecessary.

**2. src/components/auth/LoginScreen.tsx lines 38 and 40** — Remove unnecessary double casts:
Change line 38: `setSuccessUser(userInfo as unknown as UserInfo);`
To: `setSuccessUser(userInfo);`
Change line 40: `setAuth(token, userInfo as unknown as UserInfo);`
To: `setAuth(token, userInfo);`
Reason: `oauthExchangeCode` returns `Promise<UserInfo>`, matching both `setSuccessUser(UserInfo | null)` and `setAuth(string, UserInfo)`.

**3. src/routes/repo-browser/FileActions.tsx lines 118-119** — Remove dead invalidateQueries calls:
Delete these two lines entirely:
```
await queryClient.invalidateQueries({ queryKey: ['models'] });
await queryClient.invalidateQueries({ queryKey: ['datasets'] });
```
Reason: Query keys are `['repos', 'models', ...]` and `['repos', 'datasets', ...]`. The `['repos']` invalidation on line 117 already matches both via prefix matching. `['models']` and `['datasets']` alone match nothing.

**4. src/stores/uploadStore.ts** — Remove dead `wizardRepoId` and `setWizardRepoId`:
- Remove from interface: `wizardRepoId: string | null;` and `setWizardRepoId: (repoId: string | null) => void;`
- Remove from store state: `wizardRepoId: null,`
- Remove from store actions: `setWizardRepoId: (repoId) => set({ wizardRepoId: repoId }),`
Reason: These are never read or written by any component. Dead code from an unused drag-drop pre-selection feature.

**5. src/stores/uploadStore.test.ts** — Remove `wizardRepoId` from test reset state:
Change the setState call to not include `wizardRepoId: null` (it no longer exists in the interface).
