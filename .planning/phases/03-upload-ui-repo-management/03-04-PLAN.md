1. Create `src/routes/repo-browser/RepoBrowserPage.tsx`:
   - Props: `repoId: string`, `repoType: 'model' | 'dataset'`, `onBack: () => void`
   - Header: back arrow button (returns to models/datasets), repo name with type badge, "Delete Repo" danger button
   - Two tabs below header: "Files" (default) and "History" (placeholder for Plan 05)

   - Error state if fetch fails (show error message with retry button)

2. Create `src/routes/repo-browser/FileTree.tsx`:
   - Takes the flat `ListFileEntry[]` from useRepoFiles and builds a hierarchical tree structure:
     ```typescript
     interface TreeNode {
       name: string;       // "model.safetensors" or "subfolder"
       path: string;       // full path "subfolder/model.safetensors"
       type: 'file' | 'directory';
       size?: number;
       oid?: string;
       children?: TreeNode[];
     }
     ```
   - Algorithm: split each file's `path` by '/' to determine hierarchy. Group files under their parent directories. Sort: directories first (alphabetically), then files (alphabetically).
   - The tree building should be a pure function `buildTree(files: ListFileEntry[]): TreeNode[]` that can be unit tested
   - Render root-level nodes as FileTreeNode components
   - If repo is empty, show an empty state: "This repository is empty" with a link to the upload wizard

3. Create `src/routes/repo-browser/FileTreeNode.tsx`:
   - Props: `node: TreeNode`, `repoId: string`, `repoType: 'model' | 'dataset'`, `depth: number`
   - For directories: clickable row with ChevronRight icon that rotates 90deg when expanded. Shows folder icon (Folder from lucide). Indent based on `depth * 16px`.
   - For files: shows file icon (File from lucide), file name, size (formatBytes), and action buttons
   - Expand/collapse: local state `isExpanded`, toggles visibility of children
   - File action buttons (shown on hover or as an actions menu):
     a. Delete: opens confirmation AlertDialog (see Task 2)
     b. Copy HF URL: copies `https://huggingface.co/{repoType}s/{repoId}/blob/main/{path}` to clipboard using `navigator.clipboard.writeText()`
     c. Metadata popover/tooltip: shows size, OID (SHA), and path on hover or click
   - Use Tailwind: hover:bg-secondary for rows, cursor-pointer, transition-transform for chevron rotation

4. Wire RepoBrowserPage into AppShell:
   - Import RepoBrowserPage in AppShell.tsx
   - In ContentArea switch, `case 'repo-browser'`: render RepoBrowserPage with selectedRepoId, selectedRepoType props
   - The `onBack` prop navigates back to the previous section (models or datasets depending on repo type)

5. Wire repo card clicks in ModelsPage and DatasetsPage:
   - Both pages need a way to navigate to the repo browser when a repo card is clicked
   - AppShell should pass a `onRepoSelect: (repoId: string, repoType: 'model' | 'dataset') => void` callback to ModelsPage and DatasetsPage
   - When called: set selectedRepoId, selectedRepoType, and switch section to 'repo-browser'
   - Per user decision: "File browser opens when clicking on a repo card/row from the Models or Datasets pages"
