1. Create `src/routes/upload/queue/UploadQueueView.tsx`:
   - Reads jobs from `useUploadStore` and splits into three groups:
     a. Active: jobs where state is 'hashing' | 'uploading' | 'committing'
     b. Queued: jobs where state is 'pending' | 'paused'
     c. Completed: jobs where state is 'done' | 'failed' | 'cancelled'
   - Renders three collapsible sections with headers showing count:
     - "Active Uploads (N)" — expanded by default
     - "Queue (N)" — expanded by default
     - "Completed (N)" — collapsed by default, with "Clear All" button in header
   - If all sections empty, show empty state with upload icon and "No uploads yet" message + "Start Upload" button that opens wizard

2. Create `src/routes/upload/queue/ActiveJobRow.tsx`:
   - Props: `job: UploadJob`
   - Looks up progress from `useUploadStore.progressMap[job.id]`
   - Displays:
     a. File name (truncated with title tooltip)
     b. Target repo name and type badge
     c. Progress bar (shadcn Progress component) showing `bytes_sent / total_bytes` percentage
     d. Speed: `formatSpeed(progress.speed_bps)` — show "-- MB/s" if no progress yet
     e. ETA: `formatEta(progress.eta_seconds)`
     f. Bytes: `formatBytes(progress.bytes_sent) / formatBytes(progress.total_bytes)`
     g. State badge (hashing/uploading/committing) with appropriate color
   - Action buttons:
     a. Pause button (calls `pauseUpload(job.id)`)
     b. Cancel button (calls `cancelUpload(job.id)`) with confirmation
   - If state is 'paused': show Resume button instead of Pause

3. Create `src/routes/upload/queue/QueuedJobRow.tsx`:
   - Compact row: file name, repo name, priority star, actions
   - Star icon (lucide Star): filled yellow if `job.priority === true`, outline if false
   - Click star: calls `setUploadPriority(job.id, !job.priority)` then refreshes job list via `listUploads().then(setJobs)`
   - Per user decision: "click to toggle priority (moves to front of queue)"
   - Action buttons: Cancel (with confirmation), Resume (if paused)

4. Create `src/routes/upload/queue/CompletedJobRow.tsx`:
   - Compact row: file name, repo name, state badge (done=green, failed=red, cancelled=gray)
   - If failed: show `job.last_error` truncated with tooltip
   - Action: X button to remove from list (filter out from store)

5. Update `src/routes/upload/UploadPage.tsx`:
   - Wire UploadQueueView as the queue tab/view
   - Add "New Upload" button in the header that opens the wizard
   - Structure: toolbar at top (New Upload button + Pause All button), then UploadQueueView below
   - The "Pause All" button calls `pauseAllUploads()` when there are active uploads

1. Create `src/routes/upload/MiniRepoPicker.tsx`:
   - A Dialog (shadcn) that appears when files are dropped onto the app
   - Props: `open: boolean`, `onOpenChange: (open: boolean) => void`, `droppedPaths: string[]`, `onConfirm: (repoId: string, repoType: string) => void`
   - Shows a compact repo list (uses useModels + useDatasets, combined and sorted by lastModified)
   - Quick search input at top to filter repos by name
   - Each repo row: name, type badge, click to select
   - Selected repo highlighted. "Upload to [repo]" confirm button at bottom
   - Commit message input with default "Add files via Face Hugger"
   - On confirm: enqueue all dropped files to selected repo, close dialog, switch to upload section

2. Wire app-wide drag-and-drop in `src/components/shell/AppShell.tsx`:
   - Add state: `droppedPaths: string[]`, `miniPickerOpen: boolean`, `isDragOver: boolean`
   - Register `getCurrentWebview().onDragDropEvent()` in a useEffect:
     - On `type === 'enter'`: set `isDragOver = true`
     - On `type === 'drop'`: set `isDragOver = false`, set `droppedPaths = event.payload.paths`, set `miniPickerOpen = true`
     - On `type === 'leave'`: set `isDragOver = false`
   - Return unlisten function in useEffect cleanup: `unlisten.then(fn => fn())`
   - When `isDragOver` is true, show a full-app overlay with "Drop files to upload" message and an upload cloud icon (semi-transparent overlay on top of everything)
   - Render MiniRepoPicker dialog with the state
   - In MiniRepoPicker's onConfirm callback: for each path, call `enqueueUpload`, then navigate to upload section

   CRITICAL: Use `getCurrentWebview().onDragDropEvent()` from `@tauri-apps/api/webview` — NOT HTML5 ondrop. HTML5 drag events do not fire for OS file drops in Tauri desktop apps on macOS/Linux.

3. Create `src/routes/upload/FolderSync.tsx`:
   - A component accessible from UploadPage (add a "Folder Sync" tab/button)
   - Flow:
     a. User clicks "Select Folder" — calls `open({ directory: true })` from plugin-dialog
     b. User selects target repo from a repo picker (inline list, similar to wizard step 1 but compact)
     c. App compares local folder to remote repo:
        - Local: use `readDir(folderPath, { recursive: true })` from @tauri-apps/plugin-fs to list all files. Build a map of `relativePath -> {name, size}`. Get file sizes using `stat()` from @tauri-apps/plugin-fs.
        - Remote: use `listFiles({ repo, recursive: true, accessToken: token })` from @huggingface/hub. Build a map of `path -> size`.
        - Diff: file is "new" if not in remote map. File is "changed" if in remote map but different size. File is "unchanged" if same size. File is "remote only" if in remote but not local (not uploaded, just informational).
     d. Show diff table: filename, status (New/Changed/Unchanged/Remote Only), local size, remote size
     e. Default: New and Changed files are checked (selected for upload), Unchanged are unchecked
     f. "Sync N files (X MB)" button — enqueues only checked files
   - Per user decision: "one-time manual sync — no watch mode"
   - Commit message for sync: "Sync folder [folderName] via Face Hugger"

4. Add FolderSync as a tab in UploadPage:
   - UploadPage now has three views: "New Upload" (wizard), "Queue", "Folder Sync"
   - Tab bar or segmented control at the top to switch between them
